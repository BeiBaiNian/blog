智养舒居：基于YOLOv8-pose与多模态的居家安全监护系统实战

  • 2026-04-29
  • 作者 董子阳
  • ~2.69K 字

这篇博客分享一个我大二、大三期间主导并深度参与的实战项目《智养舒居—基于YOLOv8-pose构建的多模态居家安全监护系统》。

这个项目源于日益严峻的老龄化社会背景，传统设备存在盲区大、隐私泄露风险高、且误报率高的问题。为了解决这些痛点，我们团队从零开始，研发了一套结合 AI 视觉、环境感知与边缘计算的全闭环智慧系统。

写这篇博客，主要是想对这个项目做一个阶段性的技术复盘，梳理一下我在其中应用的技术栈、攻克的难点以及最终的落地成果。

🛠️核心技术栈总览

在这个项目中，我全面参与了从底层算法、硬件接入到前后端平台开发的全流程，主要技术栈涵盖：

  • 大模型的训练：YOLOv8-pose( 姿态估计 )、SVM( 支持向量机 )、K-means 聚类算法、Python。
  • 后端开发：Spring Boot 框架、RESTful API 架构、MySQL 数据库、Redis 缓存系统。
  • 前端与跨端开发：Vue.js、Element-UI、Uni-App( 微信小程序开发 )。
  • 边缘计算与硬件：Raspberry Pi( 树莓派 4B )、多源传感器网络( DHT11 温湿度、MQ-2 烟雾传感器 )、Linux 环境部署。

💡我的核心工作与难点攻克

为了让系统真正在复杂居家环境中落地，我在开发过程中投入了大量精力解决实际工程问题。以下是我在项目中主导的几个核心模块：

1. 边缘计算与硬件通信部署 (Edge Computing & Hardware)

为了在控制成本的前提下保证高可用性，系统的感知层完全基于边缘计算架构独立搭建。我负责了整个硬件节点的选型、环境配置、代码烧录与通信链路打通。

  • 硬件选型与装配：以树莓派 4B 为核心主控，通过 CSI 接口外挂 Camera Module V2 采集视频流，并利用 GPIO 引脚精准接入 DHT11（温湿度）与 MQ-2（烟雾）多源传感器。
  • 环境部署与推流：在树莓派中配置了基于 Ubuntu 的轻量级 Linux 运行环境，编写 Python 数据采集脚本，实现传感器数据的实时读取与视频流的硬件解码。
  • 数据上云：优化了数据传输协议，通过内置 Wi-Fi 模块，将预处理后的多模态数据以低于 80ms 的极低延迟稳定推流至云端服务器，打通了端到端的数据链路。

2. 核心算法调优与多模态数据融合 (AI Model & Data Analysis)

这是整个系统最核心的“大脑”部分。为了在复杂居家环境中精准识别老人行为，我主导了深度学习模型的训练与多模态数据的融合策略。

  • 大模型训练与显存优化：放弃了算力要求过高的传统方案，转而采用 YOLOv8-pose 框架。全程主导了涵盖 4 类核心动作（站、坐、蹲、躺）的 7900 张海量数据集的采集、清洗与标注。面对训练时的显存瓶颈，我设计了分批次迁移学习策略，并通过模型集成技术，最终将行为识别准确率拉升至 96.1%。
  • 多模态数据融合：单一的视觉检测极易产生误报。我引入了融合算法，将视觉提取的人体 17 个关键点坐标，与传感器传回的环境温湿度、烟雾浓度进行融合特征提取。
  • SVM 风险决策：利用 Z-score 归一化处理异构数据，并基于径向基函数（RBF）构建了 SVM 风险决策模型。实现了“异常姿态 + 环境参数”的交叉验证，构建了精准的异常行为分级预警机制。

3. 全栈架构开发与多端业务闭环 (Full-Stack & Multi-Terminal)

为了让底层数据真正服务于业务场景，我参与搭建了基于前后端分离的完整业务闭环，实现了从数据感知到用户干预的无缝衔接。

  • 后端中枢支撑：采用 Spring Boot 框架构建高并发后端，使用 MySQL 持久化结构数据，并引入 Redis 缓存层大幅提升高频请求的响应速度。
  • Web 管理端与数据大屏：使用 Vue.js 配合可视化工具，为社区工作人员开发了直观的数据可视化大屏与实时监控后台，实现了区域养老资源的全局掌控。
  • 适老化移动端：基于 Uni-App 开发了微信小程序。内嵌腾讯云 TRTC 实现低延迟音视频通话，支持语音交互智能点餐与健康档案追踪，彻底打通了“监测-预警-响应”的最后一公里。

🏆 项目完成度与阶段性成果
